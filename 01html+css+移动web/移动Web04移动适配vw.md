# vmin和vmax
- 分别取vw和vh最小和最大的那个
- 如果手机横屏了，vw和vh就变了，但是vmin和vmax就不变
# vw/vh
- 相对单位 -- 相对视口的尺寸计算结果
- 1vw = 1/100视口宽度
- 1vh = 1/100视口高度
	```html
	<style>
		.box{
			width: calc(68/3.75*1vw);
			height: calc(29/3.75*1vw);
			background-color: pink;
		}
	</style>
	```
- 如果在移动端等比例，不仅不能混用，而且正常只能是vw
	- 因为高度可以显示更多内容，高度可以适应内容，人是往下看手机的
	- 但是内容只能适应宽度，大部分时候不会横向滑动看内容
	- 也就是纵向排版的内容，只能用vw
	- 所谓纵向排版，就是纵向会溢出，横向最多占满
- 若有一天做横向排版的页面，那自然是用vh适配
# dvw/dvh
- 当前视口尺寸 -- 动态视口尺寸

|特性	|dvw（动态视口宽度）	|vw（静态视口宽度）|
|:---:|:---:|:---:|
|计算基准	|当前可视视口宽度	|固定的布局视口宽度|
|适配动态|UI（地址栏 / 虚拟键盘）|	实时适配，值随可视区域变化	|不适配，值始终固定|
|移动端布局体验	|无布局错乱，体验优	|虚拟键盘弹出时易溢出 / 滚动|
|适用场景	|移动端 H5、小程序	|PC 端 / 静态布局的移动端|
